## What the demo shows

The demo walks through all three render states of the new Dashboard + BooksShell
gate, plus the reachable routes from each:

| # | Frame | Route | State | What it shows |
|---|---|---|---|---|
| 1 | 01 | `/books` | C (business exists) | 4-link sidebar (Setup Wizard removed), status bar ("Setup ✓ Done · Categories ⚠ Not started"), "Welcome back, X2." card with 3 quick-link CTAs |
| 2 | 02 | `/books/categories` | C | Categories page still works after the sidebar change (B1 surface — unchanged) |
| 3 | 03 | `/books/transactions` | C | Transactions page still works (B1 surface — unchanged) |
| 4 | 04 | `/books/settings` | C | Settings page renders with submenu (General / Categories / Other) |
| 5 | 05 | `/books/setup` | C | Setup Wizard route still reachable by direct URL — proves the wizard wasn't removed from the app, just from the sidebar |
| 6 | 06 | `/books` | A (404 / first-run) | Full-page centered welcome card ("Welcome to Virta Books."). NO sidebar. NO status indicators. NO quick links. One CTA. |
| 7 | 07 | `/books/setup` | A | The "Set up your books →" CTA from frame 6 lands on the existing Setup Wizard stub |
| 8 | 08 | `/books/transactions` | A | First-run + direct route to non-Dashboard page → "Books not set up yet" banner with CTA, page renders below |
| 9 | 09 | `/books` | C (restored) | Sanity check — restoring the businesses table returns to State C |

## How the demo was recorded

The Vite dev server (port 5173) and the Books API (port 3001) were both
running. The businesses table was temporarily cleared (snapshot → DELETE →
recapture State A frames → INSERT snapshot back) so the 404 path of
`GET /api/v1/books/businesses/current` could be exercised. The full original
businesses table is preserved (snapshotted to `_biz_snapshot` then restored).

Frames were captured with `Chrome --headless --screenshot` at 1280×800, then
stitched into an MP4 with `ffmpeg -framerate 1/40` (40s per frame) for a
6-minute silent walkthrough.

## DB safety

- Backup: `data/tasks.db.backup-b2a-wizard-a-1783983529` (taken before the
  demo, before any mutation).
- `_biz_snapshot` was used as a temporary in-DB copy and dropped after
