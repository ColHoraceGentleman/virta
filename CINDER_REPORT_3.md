# Cinder Report #3 — Virta Task Manager: Per-Project Categories & Dark Mode

## Summary

Implemented per-project categories and per-project dark mode for the Virta task manager. The `dark_mode` column was added to the `projects` table via `ALTER TABLE` with a try/catch guard, and the `project_id` column was added to `categories` with existing rows migrated to the Personal project (ID `ca272e5f2aa23e801b54fa09e48852a7`). The backend's `createProject`/`updateProject` functions now accept and store `darkMode`, `getCategories` filters by `project_id`, and `createCategory` accepts and stores `projectId`. The client-side `useTasks` hook now manages `darkMode` state sourced from the current project's `dark_mode` field — switching projects automatically applies that project's theme, and the toolbar toggle persists to the DB. Categories are now scoped per-project: `loadCategories` passes `currentProject.id`, `createCategory` sends `projectId`, and the GET/POST category routes accept `?projectId=` or body `projectId`. SettingsModal received a full UI overhaul with collapsible sections for both projects and categories, each with an inline expand/collapse form (name + dark/light toggle for projects, name + color swatch for categories). An empty-state screen appears when zero projects exist. The build succeeded with no errors (`✓ 46 modules transformed`) and the service restarted cleanly, responding on port 3001 with the new `dark_mode` field in project payloads.
